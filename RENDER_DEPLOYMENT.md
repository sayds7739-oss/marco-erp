# Render Deployment

This repository is prepared for Render deployment using Docker.

## What was added

- `Dockerfile` for `src/MarcoERP.API`
- `.dockerignore` to keep the build context small
- `render.yaml` blueprint for a web service and PostgreSQL database
- Render runtime support in `Program.cs` for:
  - binding to Render's `PORT`
  - forwarded headers behind Render's proxy
  - optional startup migrations via `DB_APPLY_MIGRATIONS_ON_STARTUP`

## Required environment variables

These are already declared in `render.yaml`:

- `ASPNETCORE_ENVIRONMENT=Production`
- `DB_PROVIDER=PostgreSQL`
- `DB_CONNECTION` from the Render PostgreSQL service
- `DB_APPLY_MIGRATIONS_ON_STARTUP=true`
- `MARCOERP_JWT_SECRET` generated by Render
- `Jwt__Issuer=MarcoERP.API`
- `Jwt__Audience=MarcoERP.Mobile`
- `Cors__AllowedOrigins__0=https://your-frontend-domain.onrender.com`

## After creating the Render service

Update `Cors__AllowedOrigins__0` to the real frontend origin if the API will be called from a browser.

## Notes

- The API uses PostgreSQL on Render through `DB_PROVIDER=PostgreSQL`.
- Startup migrations can be disabled by setting `DB_APPLY_MIGRATIONS_ON_STARTUP=false`.
- `MarcoERP-Mobile.apk` is intentionally excluded from source control.
